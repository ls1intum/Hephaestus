The Intelligence Service provides the AI-powered mentor chat assistant. It's built with Node.js, Hono, and AI SDK v6 (Vercel).

## Architecture Overview

The service is a pure backend API that:
- Uses [Hono](https://hono.dev/) for HTTP routing with OpenAPI support
- Leverages [AI SDK v6](https://ai-sdk.dev/) (beta) for LLM interactions
- Connects to the shared PostgreSQL database via [Drizzle ORM](https://orm.drizzle.team/)
- Supports multiple LLM providers (OpenAI, Azure OpenAI)

## Local Development

### Prerequisites

- Node.js >= 22.10.0
- Access to the shared PostgreSQL database (via Docker or [dockerless setup](/contributor/dockerless-postgres))
- API keys for at least one LLM provider

### Setup

1. Navigate to the service directory:

   ```bash
   cd server/intelligence-service
   ```

2. Install dependencies:

   ```bash
   npm install
   ```

3. Configure environment variables:

   ```bash
   cp .env.example .env
   ```

   Edit `.env` with your credentials:

   ```bash
   DATABASE_URL=postgresql://root:root@localhost:5432/hephaestus
   
   # Choose one LLM provider
   OPENAI_API_KEY=sk-...
   # or
   AZURE_RESOURCE_NAME=your-resource
   AZURE_API_KEY=your-key
   
   # Model configuration (provider:model format)
   MODEL_NAME=openai:gpt-4o-mini
   ```

4. Start the development server:

   ```bash
   npm run dev
   ```

   The service runs at [http://localhost:8000](http://localhost:8000) with:
   - `/docs` – Interactive API documentation
   - `/openapi.yaml` – OpenAPI v3.1 specification

## AI Observability

The intelligence service exposes an OpenTelemetry instrumentation hook (`src/instrumentation.ts`) that is currently a no-op stub. A generic OTel exporter will be wired up alongside the same-image mentor migration; until then there is no AI tracing in this service.

The Langfuse integration was removed in the Pi-only consolidation. Prompts are now defined inline in `src/mentor/chat.prompt.ts` and compiled by the static loader in `src/prompts/loader.ts`.

## Database Access

The service connects to the same PostgreSQL database as the application server. Database schema is managed by the application server (Spring Boot/Liquibase migrations).

### Regenerating Drizzle Schema

After schema changes in the application server:

```bash
npm run db:introspect
```

This introspects the database and generates TypeScript models in `src/shared/db/schema.ts`.

## Testing

Run the test suite:

```bash
npm run test
```

Tests use Vitest and mock external AI providers to ensure deterministic results.

## Type Checking and Linting

```bash
npm run typecheck  # TypeScript compilation check
npm run check      # Biome linting and formatting
```

## OpenAPI Export

Generate the OpenAPI specification file:

```bash
npm run openapi:export
```

This writes `openapi.yaml` to the service root, which is used by the webapp's API client generator.
