## Service layout

The platform is layered into five services. See the canonical [architecture diagram](/contributor/conceptual-model#architecture) for the component view and data flows; the responsibilities are:

- **Application Server** — Spring Boot. Integrates with code platforms, owns the practice catalog and findings, exposes the primary REST API.
- **Intelligence Service** — TypeScript + Hono. Powers the AI mentor and conversational artifacts.
- **Webhook Ingest** — Event ingestion for GitHub/GitLab webhooks, decoupled from downstream processing via a message broker.
- **Agent Sandbox** — Isolated Docker containers running AI agents (Claude Code, OpenCode) for practice detection.
- **User Management** — Keycloak identity realm configured with GitHub as identity provider.

Externally Hephaestus integrates with code hosting (GitHub primary, GitLab partial), Slack notifications, and an analytics stack for anonymised usage metrics.

## AI mentor architecture

![LangGraph structure for AI mentor](./img/system-design/langgraph.png)

The mentor relies on LangGraph nodes that update storage, generate responses, and maintain state. Conditional edges decide which node executes next based on the conversation context.

Each call to the mentor executes the graph: storage update nodes persist longitudinal context, response nodes craft user-facing replies, state nodes mutate the TypedDict that represents the conversation, and conditional edges steer the flow.

## Core workflows

### Login and authorisation

![GitHub login and authorisation flow](./img/system-design/login_authorization_flow.svg)

The login flow uses GitHub OAuth via Keycloak. Successful login grants access tokens for both the webapp and application server.

### New review activity

![New review activity workflow](./img/system-design/new_review_activity.svg)

Webhook events trigger ingestion via the webhook-ingest service. The application server enriches the data, persists activity events, and updates downstream surfaces (practice findings, profile, leaderboard).
