## Before you start

- A GitHub account in the organisation your course or team uses.
- Access to [hephaestus.aet.cit.tum.de](https://hephaestus.aet.cit.tum.de). Course staff install the GitHub App for you.
- A current browser — Chrome, Firefox, Edge, Safari.

## Sign in

1. Go to [hephaestus.aet.cit.tum.de](https://hephaestus.aet.cit.tum.de).
2. Choose **Continue with GitHub** and complete the authorisation. Hephaestus asks for the smallest set of scopes that lets it read your pull requests.
3. You land in your organisation's workspace. There's no invite flow — the GitHub App handles membership.
4. Your teams and repository access sync from GitHub. You'll only see activity for repositories you already work on.

:::info Teams come from GitHub
Need to change which team or repositories you're under? Ask an organisation maintainer to update your GitHub team. Hephaestus picks it up automatically.
:::

## Find your way around

| Where | What it's for |
| --- | --- |
| **Leaderboard** | A picture of weekly review activity across the workspace. |
| **Profile** | What you've contributed and your review activity. |
| **Teams** | The same activity, scoped to a team. |
| **Mentor** | A conversation when you want one. Threads stick around. |
| **Settings** | Your synced GitHub identity and preferences. |

## Your first week

1. **Read a finding.** Open one of your recent pull requests and read what Hephaestus left you. Take what's useful. Push back on what isn't.
2. **Talk to the mentor.** Open the **Mentor** tab. Ask what to focus on this week, or talk through a review that surprised you.
3. **Look at your profile.** A small picture of where you've been spending your time.

## Keeping up

- Findings appear within a few minutes of activity on GitHub.
- A weekly Slack digest summarises team activity, if your admin has it on.
- More notifications (e.g. email) are coming.

Stuck? Drop into [GitHub Discussions](https://github.com/ls1intum/Hephaestus/discussions).
