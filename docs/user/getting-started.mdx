## Requirements before you start

- A GitHub account that belongs to the organisation your course or team uses.
- Access to [hephaestus.aet.cit.tum.de](https://hephaestus.aet.cit.tum.de) (course staff install the GitHub App for you).
- A modern browser (Chrome, Firefox, Edge, Safari).

## First-time sign-in

1. Navigate to [hephaestus.aet.cit.tum.de](https://hephaestus.aet.cit.tum.de).
2. Choose **Continue with GitHub** and complete the authorisation flow. Hephaestus only requests the minimal scopes required to read pull requests in repositories authorised by your organisation.
3. When the redirect completes you land in the single workspace associated with your organisation. There is no invite flow—the GitHub App handles membership automatically.
4. Your GitHub teams and repository permissions sync in the background. Hephaestus mirrors them, so you will only see activity for the repos you already touch on GitHub.

:::info Teams are GitHub-driven
Need to change the team or repositories you appear under? Ask an organisation maintainer to update the GitHub team membership or App installation—Hephaestus refreshes automatically.
:::

## Tour the main navigation

| Area | Why it matters |
| --- | --- |
| **Leaderboard** | Landing view. Shows review activity ranked by team and repository, reflecting the GitHub setup your organisation already maintains. |
| **Profile** | Your contribution history and achievements. |
| **Teams** | Browse team-filtered views of contribution activity. |
| **Mentor** | Open a thread with Heph to reflect on recent work, set goals, or prepare for a standup. Conversations persist in your thread history. |
| **Settings** | Review your synced GitHub identity and personal preferences. Repository access and team membership remain read-only because they come from GitHub. |

## Gain traction in your first week

1. **Read a finding.** Look for Hephaestus comments on your recent pull requests — verdict, evidence, and a recommended action. Confirm, dispute, or dismiss as you go.
2. **Start a mentor conversation.** Open the **Mentor** tab and send Heph a message — ask about your recent work, set a goal, or talk through what's blocking you. The mentor is a reflection partner, not a coding agent.
3. **Skim your activity.** The leaderboard (if your workspace enables it) shows weekly review activity. It's a check-in surface, not a ranking — your findings are where the actionable signal lives.

## Staying informed

- **Findings** appear as PR/MR comments and inline diff notes typically within a few minutes of GitHub events.
- **Weekly Slack digests** (if configured by your admin) highlight standout contributors and notable work each week.
- Planned features such as email digests are on the roadmap but not yet available in production deployments.
